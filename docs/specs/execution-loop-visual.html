1. The Problem: Open Loop vs Closed Loop

Trader clicks Enter → Order placed
→ → ? ? ?
Trader clicks Refresh → Maybe sees position
Risk engine blind between clicks. AutoPilot adjusts phantom positions. Kill switch can't fire.

PROPOSED — Closed Loop
Tick arrives → P&L recalculated
Risk evaluated → Conditions checked
Adjustment fired → Fill verified
Every 5 seconds: sync → verify → recalculate → evaluate → push. Autonomous.

6. What Each Phase Unlocks

Without the loop
  • Kill switch fires on stale data (or not at all)
  • AutoPilot adjusts phantom positions
  • Order fails silently — trader doesn't know
  • Greeks show entry-time values, not live
  • Risk state never transitions automatically
  • Frontend shows last-refresh data
With the loop
  • Kill switch fires within 5s of drawdown breach
  • Positions verified against broker every 30s
  • Failed orders detected in <30s, flagged to user
  • Greeks refresh every 60s with live underlying
  • Risk auto-transitions: NORMAL → DEGRADED → PAUSED
  • Frontend updates every 5s via WebSocket push
